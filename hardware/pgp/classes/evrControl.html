EvrControl¶

EvrControl objects in C++ are referenced by the following shared pointer typedef:

typedef std::shared_ptr<rogue::hardware::pgp::EvrControl> rogue::hardware::pgp::EvrControlPtr¶

Alias for using shared pointer as EvrControlPtr.

The class description is shown below:

class EvrControl : public PgpEvrControl¶

EVR Control Class.

This class contains the current EVR configuration for one of the 8 lanes on a PGP card. This class is used to obtain the current configuration of the EVR functions of the PGP lane and to then modify the configuration. There is one member, evrEnable, which is global to the entire card. Updating this bit will impact the operation of the other 7 channels. This class is a C++ wrapper around the PgpEvrControl structure used by the lower level driver. All structure members are exposed to Python using their original names and can be read and updated directly.

The structure which the above class wraps is shown below. All of these members are exposed in the above class.

struct PgpEvrControl¶

EVR Control, one per lane.

Subclassed by rogue::hardware::pgp::EvrControl

Public Members

uint32_t lane¶

Lane number associated with this record.

uint32_t evrEnable¶

Global EVR enable for all lanes, 1 = enable, 0 = disabled.

uint32_t laneRunMask¶

Run trigger enable for this lane, 1 = enable, 0 = disable.

uint32_t evrSyncEn¶

EVR Sync enable, 1 = start, 0 = stop.

uint32_t evrSyncSel¶

Sync select, 0 = async, 1 = sync for start/stop.

uint32_t headerMask¶

Header checking mask, 1 enable bit for each of 4 virtual channels.

uint32_t evrSyncWord¶

EVR Sync word, 32-bit timing fidicial to transition start/stop on.

uint32_t runCode¶

8-bit timing code to assert run trigger

uint32_t runDelay¶

Delay between timing code reception and assertion of run trigger.

uint32_t acceptCode¶

8-bit timing code to assert accept trigger

uint32_t acceptDelay¶

Delay between timing code reception and assertion of accept trigger.
