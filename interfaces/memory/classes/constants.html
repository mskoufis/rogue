Constants¶

The following constants are defined for use with the memory interface.

Transaction Types¶

The following memory transaction types are defined:

const uint32_t rogue::interfaces::memory::Read = 0x1¶

Memory read transaction.

Exposted to python as rogue.interfaces.memory.Read

const uint32_t rogue::interfaces::memory::Write = 0x2¶

Memory write transaction.

Exposted to python as rogue.interfaces.memory.Write

const uint32_t rogue::interfaces::memory::Post = 0x3¶

Memory posted write transaction.

Exposted to python as rogue.interfaces.memory.Post

const uint32_t rogue::interfaces::memory::Verify = 0x4¶

Memory verify readback transaction.

Exposted to python as rogue.interfaces.memory.Verify

Transaction Errors¶

The following transaction errors are defined:

const uint32_t rogue::interfaces::memory::TimeoutError = 0x01000000¶

This is a software level timeout that is triggered if the memory Slave never completed the transaction.

Exposted to python as rogue.interfaces.memory.TimeoutError

const uint32_t rogue::interfaces::memory::VerifyError = 0x02000000¶

The readback verify value did not match the written value.

Exposted to python as rogue.interfaces.memory.VerifyError

const uint32_t rogue::interfaces::memory::AddressError = 0x03000000¶

The Slave has declared an address error. This can occur if the address is at an alignment not support by the memory Slave

Exposted to python as rogue.interfaces.memory.AddressError

const uint32_t rogue::interfaces::memory::BusTimeout = 0x04000000¶

A bus timeout is declared at the hardware level indicating the bus transaction has timed out.

Exposted to python as rogue.interfaces.memory.BusTimeout

const uint32_t rogue::interfaces::memory::BusFail = 0x05000000¶

A bus failure is declared at the hardware level.

Exposted to python as rogue.interfaces.memory.BusFail

const uint32_t rogue::interfaces::memory::Unsupported = 0x06000000¶

The transaction is not unsupported.

Exposted to python as rogue.interfaces.memory.Unsupported

const uint32_t rogue::interfaces::memory::SizeError = 0x07000000¶

The Slave has declared a size violation. This can occur when the requested transaction size exceeds the maximum transaction supported by the Slave.

Exposted to python as rogue.interfaces.memory.SizeError

const uint32_t rogue::interfaces::memory::ProtocolError = 0x08000000¶

The Slave has declared a protocol error. This can occur if the transaction data was corrupted while the transaction was in progress.

Exposted to python as rogue.interfaces.memory.ProtocolError
